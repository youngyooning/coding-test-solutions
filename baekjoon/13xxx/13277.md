# 🧩 13277 큰 수 곱셈
- [문제 바로가기](https://www.acmicpc.net/problem/13277)
- **난이도** : 브론즈 5
- **알고리즘 분류** : 수학, 사칙연산, 임의 정밀도 / 큰 수 연산
- **언어** : Java 11

## 🔎 문제 분석
**문제 요약**
- 매우 큰 정수 `A`와 `B`의 곱을 구하는 문제

**입력**
- 첫째 줄 : 공백으로 구분된 `A`, `B`

**입력 조건**
- `A`, `B` : 300,000자리를 넘지 않는 0보다 크거나 같은 정수

**출력**
- `A`와 `B`의 곱

**제한 사항**
- 입력 크기가 `long`을 초과

## ♟️ 접근
최대 300,000자리의 정수가 들어올 수 있기 때문에 일반 정수 타입 사용 불가
- `BigInteger` 사용

### 💡 핵심 로직
`BigInteger` 곱셈 연산 메서드 : multiply
- `A.multiply(B)`

### ✍🏻 구현
1. `BigInteger`로 `A` 입력
2. `BigInteger`로 `B` 입력
3. `A * B` 계산 후 출력

## 👩🏻‍💻 코드
```java
import java.io.*;
import java.util.*;
import java.math.*;

public class Main {
	public static void main(String[] args) throws IOException {
		BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
		StringTokenizer st = new StringTokenizer(br.readLine());
		
		BigInteger A = new BigInteger(st.nextToken());
		BigInteger B = new BigInteger(st.nextToken());
		
		System.out.println(A.multiply(B));
	}
}
```

## 🍯 해결 팁
- 입력 자리수가 비정상적으로 크고 큰 수 연산, 팩토리얼, 조합 계산처럼 오버플로우 위험이 있을 때 `BigInteger`를 고려한다.
- `10 * 10`처럼 단순 계산에도 `BigInteger`를 사용하면 불필요한 오버헤드가 생길 수 있으므로 단순 계산에는 사용하지 않는 것이 좋다.
